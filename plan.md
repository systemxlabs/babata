# Agent 管理界面开发计划

## 目标
为 Babata Web 界面添加 Agent 管理功能，让用户可以查看和管理已配置的 Agents。

## 当前状态
- 已有 Dashboard 首页展示统计信息
- 已有 Tasks 任务管理页面
- 已有 `/api/agents` 接口可以获取 Agent 列表

## 执行计划

### 步骤 1: 设计导航结构
**描述**: 在 App.tsx 中添加侧边栏导航，支持在 Dashboard、Tasks、Agents 页面之间切换
**验收标准**:
- [ ] 侧边栏导航组件正常运行
- [ ] 可以在 Dashboard、Tasks、Agents 三个页面之间切换
- [ ] 当前选中页面有高亮显示

### 步骤 2: 创建 Agent 类型定义和 API
**描述**: 完善 Agent 相关类型定义和 API 函数
**验收标准**:
- [ ] Agent 类型定义完整（包含 name, description）
- [ ] 已存在 `api.getAgents()` 可用

### 步骤 3: 创建 Agent 管理页面组件
**描述**: 创建 `pages/Agents/Agents.tsx` 组件，展示 Agent 列表
**验收标准**:
- [ ] Agent 列表以卡片或表格形式展示
- [ ] 显示 Agent 名称和描述
- [ ] 显示 Agent 总数统计
- [ ] 页面有加载状态和错误处理
- [ ] 样式与现有页面保持一致

### 步骤 4: 集成导航和路由
**描述**: 在 App.tsx 中集成导航，根据当前选中项显示对应页面
**验收标准**:
- [ ] 导航切换正常
- [ ] 默认显示 Dashboard 页面
- [ ] 各页面独立渲染，状态不互相干扰

### 步骤 5: 验证和构建
**描述**: 验证所有功能正常运行，构建成功
**验收标准**:
- [ ] `npm run build` 构建成功无错误
- [ ] Agent 页面能正常显示所有 Agents
- [ ] 导航切换流畅

## 依赖关系
```
步骤 1 → 步骤 4
步骤 2 → 步骤 3
步骤 3 → 步骤 4
步骤 4 → 步骤 5
```

## 并行性分析
- 步骤 1 和步骤 2/3 可以并行执行
