# 任务管理界面设计文档

## 1. 需求分析

### 功能需求
- **根任务列表**: 展示所有根任务，支持分页/滚动加载
- **任务树展示**: 以树形结构展示任务及其子任务
- **任务详情弹窗**: 点击任务查看详细信息
- **任务目录**: 展示任务目录下的文件列表
- **任务日志**: 展示任务执行日志
- **删除功能**: 根任务列表支持删除操作
- **状态筛选**: 根据任务状态（运行中/已完成/失败/已暂停/已取消）筛选

### 页面结构
```
任务管理页面
├── 顶部筛选栏
│   ├── 状态筛选（全部/运行中/已完成/失败/已暂停/已取消）
│   └── 刷新按钮
├── 主体区域（左右布局）
│   ├── 左侧：根任务列表
│   │   ├── 任务卡片（可展开）
│   │   │   ├── 任务基本信息
│   │   │   └── 删除按钮
│   │   └── 子任务树（展开后显示）
│   └── 右侧：任务详情面板
│       ├── 基本信息
│       ├── 任务目录
│       └── 任务日志
└── 或：任务详情弹窗（移动端适配）
```

## 3. 数据模型

### Task 类型
```typescript
interface Task {
  task_id: string;
  description: string;
  agent: string;
  status: 'running' | 'completed' | 'failed' | 'paused' | 'canceled';
  parent_task_id: string | null;
  root_task_id: string | null;
  created_at: string;
  never_ends: boolean;
}
```

### TaskTree 类型（前端组装）
```typescript
interface TaskTreeNode extends Task {
  children: TaskTreeNode[];
  level: number;
}
```

## 7. 技术要点

### 任务树构建算法
```typescript
function buildTaskTree(tasks: Task[]): TaskTreeNode[] {
  const taskMap = new Map<string, TaskTreeNode>();
  const roots: TaskTreeNode[] = [];
  
  // 第一遍：创建节点
  tasks.forEach(task => {
    taskMap.set(task.task_id, { ...task, children: [], level: 0 });
  });
  
  // 第二遍：建立父子关系
  tasks.forEach(task => {
    const node = taskMap.get(task.task_id)!;
    if (task.parent_task_id && taskMap.has(task.parent_task_id)) {
      const parent = taskMap.get(task.parent_task_id)!;
      parent.children.push(node);
      node.level = parent.level + 1;
    } else {
      roots.push(node);
    }
  });
  
  return roots;
}
```
