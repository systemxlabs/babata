## Summary
Flatten the message store schema by adding dedicated columns for each message field and add `created_at` timestamp to all Message enum variants.

## Changes

### Message enum (src/message.rs)
- Add `created_at: DateTime<Utc>` field to all variants:
  - `UserPrompt`
  - `UserSteering`
  - `AssistantResponse`
  - `AssistantToolCalls`
  - `ToolResult`
- Add `Message::created_at()` getter method
- Add `MessageRecord` struct for database mapping with columns:
  - `id`, `message_type`, `content`, `reasoning_content`
  - `tool_calls`, `call_id`, `tool_name`, `args`, `result`
  - `created_at`
- Add `MessageType` enum for type discrimination

### Message store (src/memory/store.rs)
- Update table schema with flattened columns
- Add `message_to_record_fields()` helper to convert Message to DB fields
- Add `record_to_message()` helper to convert DB record to Message
- Update `append_messages()` to use INSERT with individual columns
- Update `scan_recent_messages()` to reconstruct Message from columns

### Dependencies
- Enable `serde` feature for `chrono` crate

### Updated files
- `src/agent/runner.rs` - Add created_at to Message creation
- `src/provider/anthropic_compatible.rs` - Add created_at to Message creation
- `src/provider/openai_compatible.rs` - Add created_at to Message creation
- `src/memory/mod.rs` - Update tests

## Testing
- ✅ All 115 tests pass

## Breaking Change
This change modifies the database schema. Existing databases with the old JSON message column will need to be recreated.
